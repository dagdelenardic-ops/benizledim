# Kimi - Yeni Ödev Paketi

## Rolün

Sen bu fazda **Frontend (Vue.js) & UI/UX Geliştirme Uzmanı**sın.

## Hedef

Özellikle Admin Panel ve Authentication (Giriş/Kayıt/OAuth) ekranlarındaki kullanıcı deneyimini iyileştirmek, API entegrasyonlarını sorunsuz hale getirmek ve arayüz hatalarını gidermek.

## Zorunlu Kurallar

1. Önce projeyi kavramak için `CLAUDE.md` ve güncel controller (özellikle `Auth` klasörü) dosyalarını oku.
2. Genellikle `resources/js/**` altındaki Vue.js/JS ve CSS/Tailwind dosyalarında çalış.
3. Var olan backend logic’ini bozmadan, frontend tarafındaki hataları (konsol hataları, state sorunları) çözerek ilerle.
4. Küçük commitler at: `feat(ui):`, `fix(admin):`, `style(components):`.

## Görevler

### 1) Admin Login ve Google OAuth UI İyileştirmeleri

- Giriş formunda (Email/Şifre) ve "Google ile Giriş Yap" butonlarında loading (yükleniyor) state'lerini yönet.
- Hatalı girişlerde (örn. yanlış şifre, 401 Unauthorized, 429 Too Many Requests) kullanıcı dostu bildirimler (Toast/Snackbar) göster.
- Yönlendirme (Redirect) mantığını Vue Router/PWA tarafında pürüzsüz hale getir.

### 2) Session ve API Hata Yönetimi

- Axios / Fetch için global bir interceptor kontrolü eklenecek.
- Eğer API `401` veya `419` (CSRF token mismatch) dönerse, kullanıcıyı otomatik olarak session süresinin dolduğuna dair uyarıp `/login` sayfasına yönlendir.
- Sekmeler arası oturum kontrolü (bir sekmede çıkış yapılınca diğer sekmede de oturumun kapatılması) eklenecek.

### 3) Admin Dashboard & Yetkilendirme (Role-Based UI)

- Giriş yapan kullanıcının rolüne (Admin, Editor, Author vb.) göre Admin Panel yan menüsünü (Sidebar) dinamik olarak filtrele.
- Kullanıcı avatarını ve bilgilerini Header kısmına entegre et.
- Mobil cihazlarda (Responsive) Admin Panel düzeninin (Sidebar aç/kapat) sorunsuz çalıştığından emin ol.

### 4) Performans ve Kod Düzeni

- Kullanılmayan (dead code) Vue component'lerini ve CSS sınıflarını temizle.
- UI bileşenlerini (Input, Button, Modal) tekrarlayan kodlar yerine yeniden kullanılabilir (reusable) Vue Component'leri haline getir.

## Teslim Kriteri

- `npm run build` (veya Vite build) başarılı bir şekilde, hatasız ve minimum warning ile tamamlanmalı.
- Login -> Dashboard -> Logout akışı sıfır konsol hatası ile çalışmalı.
- Tüm form doğrulama (validation) süreçleri frontend tarafında da (backend'e gitmeden önce) yakalanabilmeli.
